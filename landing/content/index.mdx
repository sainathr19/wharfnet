# Wharfnet

**One-command localnet for EVM, Solana & Starknet** — boot every chain your app
touches with a single command, fund accounts from a unified faucet, and get
standard test tokens deployed at known addresses, then wire it straight into your
integration tests and CI pipelines.

<BootDemo />

`wharfnet` is the local harbor for your chains. It packages the stitching that
cross-chain and multi-VM teams do by hand — an EVM node, a Solana validator, a
Starknet devnet, plus a homemade faucet and glue scripts — into one opinionated,
reproducible tool.

## Using wharfnet locally

A single `wharfnet up` boots an EVM chain, a Starknet chain, and a Solana chain
side by side — each with funded dev accounts, test tokens at fixed addresses, and
a built-in block explorer:

```sh
wharfnet up            # chains + a block explorer each
wharfnet up --bare     # just the chains
wharfnet status        # what's running, with RPC URLs and accounts
```

Everything is deterministic by default, so tests and CI get an identical
environment every run — with opt-in persistence when you want to pick up where
you left off.
